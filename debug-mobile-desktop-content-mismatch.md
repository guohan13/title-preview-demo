# [OPEN] mobile-desktop-content-mismatch

## 问题描述
- 现象：手机端显示的内容和电脑端不一致。
- 目标：确认控制页上报、服务端保存、手机页拉取与手机页应用是否使用了同一份状态。

## 已知环境
- 项目目录：`/Users/bytedance/Desktop/design demo`
- 控制页：`index.html`
- 手机页：`mobile-preview.html`
- 同步服务：`sync_server.py`

## 初始假设
- H1：电脑端修改后没有把最新状态成功 `POST` 到 `/api/state`。
- H2：服务端保存了最新状态，但手机端没有继续轮询到它。
- H3：手机端拉到了新状态，但因为本地编辑态或其他条件，没有应用到界面。
- H4：电脑端和手机端展示的字段不是同一组，例如标题文本或阈值来源不同。
- H5：页面缓存了旧脚本，导致电脑端和手机端跑的是不同版本逻辑。

## 调试计划
1. 给控制页上报、服务端返回、手机页应用增加最小取证日志。
2. 复现“电脑端与手机端内容不一致”。
3. 基于日志确认是上报、持久化、拉取还是应用阶段出问题。
4. 再做最小修复。
