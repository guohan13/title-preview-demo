# [OPEN] shrink-level-mismatch

## 问题描述
- 现象：手机端显示为 `2档`，电脑端控制面板显示为 `1档`。
- 目标：确认两端读取的是不是同一份状态，以及是哪一段把 `maxShrinkLevels` 弄成了不一致。

## 已知环境
- 项目目录：`/Users/bytedance/Desktop/design demo`
- 控制页：`index.html`
- 手机预览页：`mobile-preview.html`
- 同步服务：`sync_server.py`

## 初始假设
- H1：控制页初始 DOM 默认值是 `1档`，但没有在加载时正确拉取服务端状态，所以一直显示本地默认值。
- H2：手机页正确拉取了服务端状态，而控制页没有把服务端状态回填到面板控件上。
- H3：控制页会在初始化时用本地默认值覆盖服务端状态，导致两端短暂或持续不一致。
- H4：服务端 `.preview_state.json` 当前存的是 `2档`，而控制页只是没有刷新或没有应用该值。
- H5：手机页和控制页访问的不是同一个状态源，导致各自显示不同。

## 调试计划
1. 给控制页、手机页、服务端增加最小取证日志。
2. 复现“电脑 1档 / 手机 2档”。
3. 比对控制页读取、服务端返回、手机页读取三段证据。
4. 基于证据做最小修复。
